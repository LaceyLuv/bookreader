# Reader TXT And EPUB Typography Plan

Status: completed/superseded by current implementation as of 2026-05-19.

The original plan covered two issues:

- TXT rendering should use display fragments even when transforms are off.
- EPUB typography controls should apply consistently to chapter content.

## Current State

- TXT rendering now uses the segment-window normalization path and measured pagination.
- EPUB typography CSS is generated by `frontend/src/lib/epubTypography.js`.
- EPUB chapter HTML is sanitized by `frontend/src/lib/epubSanitizer.js` before visible rendering and offscreen measurement.
- `frontend/src/components/EpubReader.jsx` injects reader typography CSS for both actual render and measurement paths.

## Current Validation

From `frontend/`:

```powershell
cmd /c npx vitest run src/lib/epubTypography.test.js src/lib/epubSanitizer.test.js
cmd /c npx vitest run src/components/TxtReader.segmented.test.jsx
cmd /c npm run test
```

Future typography work should update the utility tests first, then verify TXT and EPUB reader behavior through the full suite.
